# AI 文章改寫助手

## 功能概述

AI 文章改寫助手是一個 Chrome 擴展，專為提升文章寫作效率而設計。它利用先進的 AI 技術，幫助用戶快速改寫和優化文本內容，特別適合金融研究報告的撰寫。

### 產業關係圖（relationship-map）版型自適應更新

- 力模擬佈局改為視窗感知：窄畫面優先往上下展開、寬畫面優先往左右展開。
- 核心節點由固定圓形改為橢圓錨點分佈，依寬高比自動調整。
- `x/y` 力強度改為非對稱，避免某一軸空間明顯浪費。
- 視窗尺寸跨過版型門檻（portrait / balanced / landscape）時自動重排，不只做縮放。
- 圖例框已納入避障目標，節點會在模擬中主動避開圖例區域，降低遮擋機率。
- 關係圖邊界計算已納入節點文字寬度（含長英文名稱），避免窄畫面下右側節點與標籤被裁切。

### 自適應佈局系統（2026-02-13）

- **目標精簡後的主要約束**：
  1. **不出界**：完整視覺範圍（含光暈、標籤）皆在視窗內，邊界安全距離 12px
  2. **盡量填滿空間**：覆蓋率目標約 65%，由後處理歸一化與動畫期視窗自適應共同達成
  3. **避開圖例框**：圖例避障力 + refineLayout 精修流程
  4. **核心依畫面比例散佈**：橫畫面偏扁、直畫面偏長
  5. **連線長度約主體尺寸 2 倍**：連線距離公式為 `(sr + tr) × LINK_BODY_RATIO`
- **三層保障機制**：
  - 第一層：固定 tick 預模擬（先收斂，不做二次調參）
  - 第二層：後處理歸一化（等比縮放座標 + 碰撞修正）
  - 第三層：邊界強制約束（硬性限制節點視覺範圍在視窗內）
- **座標一致性修正（2026-02-13）**：歸一化後 `targetX/targetY` 會同步為最終座標，避免 fit 依據的最終位置與動畫錨點不一致，導致動畫過程再次出界。
- **自適應縮放閉迴路（2026-02-13）**：動畫期間每數個 tick 重新量測當前節點視覺邊界並平滑更新 zoom transform，持續逼近「完整在畫面內 + 儘量放大」目標，而非只在開場計算一次。
- **核心最佳散佈力（2026-02-13）**：新增 `corePacking` 力，會把核心節點放到「畫面矩形扣除圖例框」後的可用不規則區域，使用候選點最大最小距離策略自動分散，避免橫畫面卻排成上中下。
- **核心全域散佈最佳化（2026-02-13）**：核心選點改為「最小距離 + 平均距離 + 過遠懲罰 + 局部換點優化」，並以畫面長寬比作為主要形狀目標；核心錨點僅在 dirty 事件（resize/圖例更新/初始）時重算，避免週期性抖動。
- **畫面比例導向形狀（2026-02-13）**：核心分佈新增長寬比目標函數，橫畫面偏扁形展開、直畫面偏長形展開，避免所有核心集中在中區。
- **力衝突處理（2026-02-13）**：核心錨點重算後會同步更新全節點 `targetX/targetY`，避免 `corePacking` 與 `x/y` 目標力互拉；`center` 只做輕量收斂。
- **效能影響極低**：移除預模擬期間的多重自動調參後，計算路徑更短、行為更穩定

### 主要功能

1. **智能文本改寫**
   - 選取改寫：直接改寫反白的文字內容
   - 全文改寫：未選取文字時改寫整篇文章
   - 智能改寫：雙擊時自動識別特殊格式文本
   - 多任務處理：支持同時進行多個改寫任務
   - 位置追蹤：自動調整改寫位置，確保多個改寫同時進行時的準確性

2. **關鍵要點總結**

   - 位置記憶：記住上次視窗位置
   - 展開/收合：一鍵切換總結視窗顯示狀態

3. **特殊文本智能識別**
   - 雙擊時自動識別股票代碼
   - 智能保留數據和重要資訊
   - 支持自定義特殊文本規則

4. **便捷操作模式**
   - 反白文字 → 點擊改寫
   - 雙擊觸發智能改寫
   - 支持快捷鍵操作
   - 拖曳調整總結視窗位置

5. **多模型支持與自定義**
   - 支持 Gemini Pro
   - 支持 OpenAI GPT-4o
   - 可自定義 API 參數
   - 自定義改寫/總結指令模板

6. **金融專業功能**
   - 股票代碼智能識別
   - 自動補充股票資訊
   - 專業術語庫支援
   - 智能季度文字更新

7. **股票自動爬蟲功能**
   - 自動爬取台股清單：支援37個產業的完整股票清單
   - 智能合併算法：保留現有匹配規則，更新公司名稱，按代碼排序
   - 定時爬取：可設定自動爬取間隔（分鐘為單位）
   - 立即爬取：支援手動立即執行爬取
   - 實時進度：顯示爬取進度和狀態資訊
   - 資料來源：goodinfo.tw 各產業股票清單
   - 爬取控制：支援隨時停止和重新開始
   - 網頁延遲：每個網頁間隔5秒爬取，避免過於頻繁請求

**手動替換高亮框滾動修復 (2025-06-05)**
- **問題**：手動替換的高亮框在滾動後會消失，需要更新替換文字才會重新顯示

- **解決方案**：
  - **移除 IntersectionObserver**：完全移除 IntersectionObserver 機制

  - **位置計算修正**：使用相對位置 `top - scrollTop` 確保元素正確定位
  - **緩衝區優化**：在可視區域上下各200px範圍內預載入元素
- **影響**：徹底解決高亮框滾動消失問題，提升手動替換功能的穩定性

### 修復：Google 翻譯錯誤處理問題

**問題描述**：
- Google 翻譯 API 發生錯誤（如 429 配額超限）時，按鈕停留在「取消」狀態
- 計時器繼續運行，用戶無法再次點擊按鈕
- 沒有顯示友善的錯誤訊息

**根本原因**：
- `processNextBatch()` 的 catch 區塊只記錄錯誤，沒有完整清理翻譯狀態
- 未清除計時器（timeoutId）
- 未重置按鈕狀態和翻譯變數

**解決方案**：
- 在錯誤處理中添加 `clearTimeout(this.timeoutId)` 停止計時器
- 根據錯誤類型（429配額/401認證/其他）顯示友善的錯誤訊息
- 呼叫 `resetTranslation()` 完整重置翻譯狀態和按鈕
- 確保用戶能夠在錯誤後繼續使用功能

**影響範圍**：
- 檔案：`google_translator/google-translate.js`
- 方法：`processNextBatch()`
- 改善使用者體驗，特別是在 API 配額用完時

### 修復：Popup 關閉後同步停止問題

之前當 popup 頁面關閉後，自動同步功能就會停止工作，因為 SettingsIO 實例是在 popup 環境中創建的。

   - 實現完整的同步消息處理邏輯，支援所有同步操作
   - 創建持久的 SettingsIO 實例，不受 popup 生命週期影響
